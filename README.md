**Лабораторная работа №4**

**Донской А.Е. группа М8О-101БВ-25.**

**1. Цель работы:**

Освоить основы непрерывной интеграции и доставки (CI/CD) с использованием GitLab CI/CD. Научиться создавать пайплайны для автоматической сборки, тестирования и публикации Docker-образов.

**2. Задачи:**

- Создать CI/CD pipeline с автоматическим тестированием
- Настроить сборку Docker образа Python приложения
- Реализовать публикацию образа в GitLab Container Registry
- Использовать артефакты и зависимости между этапами

**3. Выполнение работы:**

1. **Создан CI/CD pipeline** с тремя этапами: test, build, deploy
2. **Этап тестирования:** автоматический запуск unit-тестов Python приложения
3. **Этап сборки:** создание Docker образа с использованием Docker-in-Docker
4. **Этап публикации:** загрузка образа в GitLab Container Registry
5. **Этап деплоя:** симуляция развертывания приложения

**4. Технологии:**

- GitLab CI/CD
- Python 3.12 + Flask
- Docker + Docker-in-Docker
- GitLab Container Registry
- Pytest для тестирования

**5. Файлы проекта:**

- .gitlab-ci.yml - конфигурация CI/CD пайплайна

**6. Pipeline этапы:**

1. **run-tests:** тестирование приложения, генерация отчета JUnit
2. **build-docker:** сборка Docker образа с тегированием
3. **publish-to-registry:** публикация образа в GitLab Registry
4. **deploy-app:** симуляция процесса развертывания

**7. Выводы:**

Освоены практические навыки настройки CI/CD pipeline в GitLab, включая автоматическое тестирование, сборку Docker образов и публикацию в container registry. Реализован полноценный пайплайн для непрерывной интеграции и доставки Python приложения.
